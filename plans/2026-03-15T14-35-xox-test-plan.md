# XOX Test Plan

## Context

XOX has no test framework configured. All quality assurance
is manual browser testing. The `docs/testing-plan.md` doc
identifies four priority areas where automated tests would
catch bugs that currently require a human to find. This plan
turns that document into an executable test strategy.

## Framework Setup

**Vitest** + React Testing Library + jsdom.

### Dependencies to install

```
npm install -D vitest @testing-library/react \
  @testing-library/jest-dom @testing-library/user-event \
  jsdom @vitejs/plugin-react
```

### Files to create

- `vitest.config.ts` -- Vitest config with jsdom, React
  plugin, `@/` path alias, setup file
- `src/__tests__/setup.ts` -- imports jest-dom matchers and
  CompressionStream polyfill
- `src/__tests__/helpers/compression.ts` -- polyfills
  `CompressionStream`/`DecompressionStream` from
  `node:stream/web` into `globalThis` (uses real deflate-raw,
  not a mock)

### package.json changes

```json
"test": "vitest run",
"test:watch": "vitest"
```

### CLAUDE.md update

Add `npm test` to the commands section.

**A. Round-trip fidelity (5 tests)**

2. Config with all steps `'1111111111111111'`
3. Config with mixed mixer states (muted, soloed, gains)
4. Config with edge BPM values (20, 300)

7. Random non-base64 string -- falls back
8. Valid base64 but invalid JSON -- falls back
9. Valid JSON `{}` -- merges with defaults
10. Valid JSON `null` -- returns defaults
11. Extra unknown fields -- ignored, known preserved
12. Wrong `version` number -- still decodes
13. Partial config (only kitId) -- other fields default
14. Missing tracks in steps -- filled from default
15. Missing tracks in mixer -- filled from default

**C. Field-level validation (14 tests)**

*validateKitId:*
16. `"808"` passes
17. `"electro"` passes
18. `"unknown"` falls back to `"808"`
19. Non-string falls back to `"808"`
20. `null` falls back to `"808"`

*validateBpm:*
21. `110` passes
22. `10` clamped to `20`
23. `500` clamped to `300`
24. `120.7` rounded to `121`
25. `NaN` defaults to `110`
26. `Infinity` defaults to `110`
27. Non-number defaults to `110`

*validateSteps:*
28. Valid 16-char binary passes
29. Length 15 uses fallback
30. Non-binary chars uses fallback
31. `null` steps uses entire fallback

*validateMixer:*
32. `gain: -0.5` clamped to `0`
33. `gain: 1.5` clamped to `1`
34. Non-boolean `isMuted` uses fallback

**D. Base64url encoding (3 tests)**
35. No `+`, `/`, or `=` in output
36. Deterministic (same input -> same output)
37. Round-trips through encode/decode layer

**E. Golden-file tests (3 tests)**
38. `defaultConfig()` snapshot
39. Known config -> known hash (catches compression
    format changes like `deflate` vs `deflate-raw`)
40. Known hash -> known config (paired with above)

**F. TRACK_IDS snapshot (1 test)**
41. `TRACK_IDS` value matches snapshot (catches
    reordering which would corrupt serialized URLs)

### Mocking

Only `CompressionStream` polyfill -- no other mocks needed.
Uses real deflate-raw compression so golden-file tests
genuinely protect against format changes.

## Key Technical Decisions

**Real compression, not mocks.** The CompressionStream
polyfill uses Node.js's real `deflate-raw` implementation.
Golden-file tests will catch if someone changes the
compression format, which would silently break every
shared URL in the wild.

**No production code refactoring for testability.** The
`handleStep` callback reads from refs inside useCallback.
Rather than extracting it into a pure function (a
production code change), the plan tests it indirectly
through the context + mocked AudioEngine. If this proves
too brittle, extracting a pure
`resolveStepSounds(trackStates, pattern, step)` function
is the recommended follow-up.

**No Playwright yet.** Unit + integration tests cover all
logic paths. Playwright for smoke testing (load, play,
export, import) is a good follow-up but lower ROI than
getting the unit layer solid first.
