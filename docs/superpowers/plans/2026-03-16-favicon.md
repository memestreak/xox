## Chunk 1: Generate Favicon Assets

### Task 1: Create SVG source

**Files:**
- Create: `src/app/favicon.svg`

- [ ] **Step 1: Create the SVG file**

The SVG uses a rounded-rect background (#0a0a0a) with
"xox" as outlined `<path>` elements (#ffffff). The
viewBox is 128x128 for clean scaling. Text paths are
pre-converted from a bold sans-serif at ~72px with
tight letter spacing.

To generate the path data, use ImageMagick to trace
bold "xox" text to SVG paths:

```bash
magick -size 128x128 xc:none \
  -font Helvetica-Bold -pointsize 72 \
  -fill white -gravity center \
  -kerning -4 \
  -annotate 0 "xox" \
  -trim +repage \
  SVG:- 2>/dev/null | grep '<path'
```

Use the extracted `<path>` elements in the SVG. If
ImageMagick's SVG output is not clean, hand-craft a
simple SVG with `<text>` converted to paths using
`magick` trace, or use a fallback approach with
`<text>` and ensure the build script handles font
rendering.

Practical fallback: since the build script uses
ImageMagick to rasterize anyway, the SVG can use
`<text>` with `font-family="sans-serif"` and
`font-weight="800"` — ImageMagick will render it
consistently on the build machine. The key constraint
is that the committed *raster* assets look correct.

```svg
<svg xmlns="http://www.w3.org/2000/svg"
     viewBox="0 0 128 128">
  <rect width="128" height="128" rx="16"
        fill="#0a0a0a"/>
  <text x="64" y="64" text-anchor="middle"
        dominant-baseline="central"
        font-family="sans-serif" font-weight="800"
        font-size="72" letter-spacing="-4"
        fill="#ffffff">xox</text>
</svg>
```

### Task 2: Create build script

**Files:**
- Create: `scripts/generate-favicon.sh`

- [ ] **Step 1: Write the build script**

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
ROOT="$(dirname "$SCRIPT_DIR")"
SVG="$ROOT/src/app/favicon.svg"
OUT="$ROOT/src/app"

# Render to 16x16 and 32x32, combine into ICO
magick "$SVG" -background none -resize 16x16 \
  /tmp/fav-16.png
magick "$SVG" -background none -resize 32x32 \
  /tmp/fav-32.png
magick /tmp/fav-16.png /tmp/fav-32.png \
  "$OUT/favicon.ico"

# Render apple-touch-icon
magick "$SVG" -background none -resize 180x180 \
  "$OUT/apple-touch-icon.png"

echo "Generated:"
echo "  $OUT/favicon.ico (16x16 + 32x32)"
echo "  $OUT/apple-touch-icon.png (180x180)"
```

- [ ] **Step 2: Make executable and run**

```bash
chmod +x scripts/generate-favicon.sh
scripts/generate-favicon.sh
```

Verify output:
```bash
magick identify src/app/favicon.ico
magick identify src/app/apple-touch-icon.png
```

Expected: favicon.ico shows 16x16 and 32x32 entries;
apple-touch-icon.png shows 180x180.

- [ ] **Step 3: Visually verify generated assets**
